# GreenWall 打包指南

本文档将指导您如何将 GreenWall 项目打包为 Windows 可执行文件 (.exe)。

## 1. 快速打包

我们为您准备了一键打包脚本。

1. 进入 `GreenWall` 目录。
2. 双击运行 `build_release.bat`。
3. 等待脚本执行完成。
4. 如果成功，生成的 `.exe` 文件将位于 `GreenWall/build/bin/` 目录下。

## 2. 环境要求

在打包之前，请确保您的开发环境已安装以下工具：

- **Go 编程语言**: (建议版本 1.18+)
- **Node.js**: (用于构建前端)
- **Wails CLI**: 项目使用的构建工具

### 安装 Wails CLI
如果您尚未安装 Wails CLI，请在终端中运行：
```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

## 3. 手动打包

如果您更喜欢使用命令行手动打包，请按照以下步骤操作：

1. 打开终端（PowerShell 或 CMD）。
2. 导航到项目根目录 `GreenWall`。
3. 运行构建命令：

```powershell
wails build -clean
```

### 命令参数说明
- `-clean`: 在构建前清理 `build/bin` 目录，确保构建纯净。
- `-platform windows/amd64`: 指定构建平台（在 Windows 上默认为当前架构，可省略）。
- `-upx`: (可选) 如果安装了 UPX，可以使用此参数压缩生成的文件大小。

## 4. 构建产物

构建完成后，您可以在以下路径找到应用程序：

```
GreenWall\build\bin\green-wall.exe
```

此文件夹中的 `.exe` 文件是独立的，可以直接发送给其他用户使用。

## 5. 常见问题

**Q: 构建时提示 "wails command not found"**
A: 请确保您已运行 `go install ...` 安装了 Wails，并且 Go 的 bin 目录已添加到系统的 PATH 环境变量中。

**Q: 构建过程中前端报错**
A: 尝试进入 `frontend` 目录，手动运行 `npm install` 确保依赖安装正确，然后再尝试构建。

**Q: 如何修改生成的 .exe 图标？**
A: 替换 `GreenWall\build\windows\icon.ico` 文件，然后重新运行构建命令即可。
